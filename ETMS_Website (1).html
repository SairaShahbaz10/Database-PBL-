ℹ️ About ETMS

PROJECT OBJECTIVES

  • 12-table relational DB for e-sports management
  • EERD with supertype/subtype (Pro/Amateur player)
  • SQL: CREATE TABLE, FK constraints, CASCADE DELETE
  • Advanced queries: JOIN, GROUP BY, HAVING, subqueries
  • Views: TeamLeaderboard & PlayerPerformance
  • Trigger: trg_AddPoints (AFTER INSERT on Matchs)
  • Full CRUD via REST API — Express.js backend
  • Referential integrity across all 12 tables

TECH STACK

  • Node.js + Express.js — REST API server
  • In-memory DB mirroring MySQL relational schema
  • 12 tables: all entities from the EERD
  • All SQL logic in server.js (backend only)
  • Vanilla HTML/CSS/JS — zero frameworks
  • Responsive sidebar + player profile drawer

BUSINESS RULES

  • BR-01 Each team must have a unique name and country.
  • BR-02 Every player must belong to exactly one team.
  • BR-03 A player is either Professional or Amateur — not both.
  • BR-04 Professional Players need a salary; Amateur Players need a school name.
  • BR-05 Each team can have only one coach.
  • BR-06 A tournament must have a prize pool before matches are scheduled.
  • BR-07 Each match must be between two different teams.
  • BR-08 Every match must have exactly one assigned referee.
  • BR-09 The winning team must be one of the two participating teams.
  • BR-10 The winning team automatically receives +3 ranking points (trigger).
  • BR-11 A sponsor can fund many tournaments; a tournament can have many sponsors.
  • BR-12 Statistics recorded only for players who participated in that match.
  • BR-13 Referee experience must be zero or greater.
  • BR-14 Deleting a team cascades to remove all related records.
